  • Endpoint /users/random
  • /api/users/random Returns a random user
  • Endpoint /users/detail/:id - returns the user details including bio
  • /api/users/detail/:id Returns user including bio.
  • Endpoint /users/:id/notes
  • /api/users/:id/notes Returns the notes from a user
  • POST /api/users/:userId/notes
    PAYLOAD { archived: boolean, text: string }
    returns the created note
  • PUT /api/users/:userId/notes/:noteId
    PAYLOAD { archived: boolean, text: string }
    returns the updated note
  • DELETE /api/users/:userId/notes/:noteId
    returns nothing
  • Endpoint /users/:id/bookmarks
  • /api/users/:id/bookmarks Returns the bookmarks from a user
  • POST /api/users/:userId/bookmarks
    PAYLOAD { category: string, url: string, rating: integer }
    returns the created bookmark
  • PUT /api/users/:userId/bookmarks/:bookmarkId
    PAYLOAD { category: string, url: string, rating: integer }
    returns the updated bookmark
  • DELETE /api/users/:userId/bookmarks/:bookmarkId
    returns nothing
  • Endpoint /users/:id/followingCompanies
  • /api/users/:id/followingCompanies Returns the companies a user is following.
  • POST /api/users/:userId/followingCompanies
    PAYLOAD { rating: number, companyId: UUID }
    returns the created following
  • PUT /api/users/:userId/followingCompanies/:followingId
    PAYLOAD { rating: number, companyId: UUID }
    returns the updated following
  • DELETE /api/users/:userId/followingCompanies/:followingId
    returns nothing
  • Endpoint /users/:id/vacations
  • /api/users/:id/vacations Returns a users vacations
  • POST /api/users/:userId/vacations
    PAYLOAD { startDate: date, endDate: date }
    returns the created vacation
  • PUT /api/users/:userId/vacations/:vacationId
    PAYLOAD { startDate: date, endDate: date }
    returns the updated vacation
  • DELETE /api/users/:userId/vacations/:vacationId
    returns nothing
  • Endpoint /companies
  • /api/companies Returns all the companies
  • Endpoint /companies/random
  • /api/companies/random Returns a random company
  • Endpoint /companies/:id/companyProfits
  • /api/companies/:id/companyProfits Returns yearly data for the company profits
  • Endpoint /products returns all the products
  • /api/products Returns all the products
  • Endpoint /offerings
  • /api/offerings Returns company offerings. Each item includes a companyId, a productId and the price of the product
  • Endpoint /users/:idx? (defaults to 0)
  • /api/users First 50 users and total count (users 1 to 50)
  • /api/users/1 Second 50 users and total count (users 51 to 100)
  • /api/users/2 Third 50 users and total count (users 101 to 150)
  • Endpoint /users/search/:term/:idx? (defaults to 0)
  • /api/users/search/Glo First 50 users who have the word Glo in their name, email or job titles and total count of users who match criteria.
  • /api/users/search/Glo/1 Second 50 users who have the word Glo in their name, email or job titles and total count of users who match criteria.
  • /api/users/search/Pro First 50 users who have the word Pro in their name, email or job titles and total count of users who match criteria.
  • /api/users/search/Pro/1 Second 50 users who have the word Pro in their name, email or job titles and total count of users who match criteria.
